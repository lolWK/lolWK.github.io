연결 리스트(LinkedList) 란?

연결 리스트, 링크드 리스트(linked list)는 각 노드가 데이터와 포인터를 가지고 한 줄로 연결되어 있는 방식으로 데이터를 저장하는 자료 구조이다. 이름에서 말하듯이 데이터를 담고 있는 노드들이 연결되어 있는데, 노드의 포인터가 다음이나 이전의 노드와의 연결을 담당하게 된다. (위키백과)

연결 리스트와 일반 배열과 다른 점은 무엇일까?


연결 리스트는 일반 배열과 다르게 데이터 요소의 선형 집합이다.
각각의 원소들은 자기 자신 다음의 원소를 가리킨다.

▶️ 연결 리스트는 순서를 표현하는 노드(node)들의 집합으로 이루어져 있다. 데이터 요소들을 노드(node)라는 객체로 구성하고, 각 노드는 데이터 필드와 다음 노드(node)를 가리키는 포인터를 가지고 있다. 이렇게 구성된 노드(node)들은 선형적으로 연결되있다.
▶️ 첫 노드(node)를 의미하는 head 포인터를 가지고 있다. 노드(node)들은 next 포인터로 연결되어있다. 연결 리스트의 마지막 노드(node)는 tail이다.
▶️ 순회하는 동안 순서에 상관없이 효율적인 삽입이나 삭제가 가능하다.
▶️ 임의 접근처럼 빠른 접근이 불가능하다.
ex) 5번째 노드(node)가 필요해도 첫 노드(node)부터 순서대로 따라가서 5번째 노드(node)로 가야한다.
▶️ 배열과 다르게 인덱스가 없다.

// 일반 배열의 경우 예시
// [12, 99, 37]

// 연결 리스트의 경우 예시.
// Node { val: 12, next: Node { val: 99, next: Node { val: 37, next: null } } },

node 만들기

node는 단일 데이터 항목을 저장하고, 호출될 다음 node들에 대한 정보인 ‘next’를 저장한다.
this.next = null 인 이유는 아직 다음 node가 없어서 null로 초기화하는 것이다.

push (끝 추가)

값을 인자로 받고, 그 값으로 새로운 node 생성
▶️ 만약에 head가 없다면 (= list가 비어있음)
=> head와 tail 모두 새롭게 생성된 node를 가리키도록 해야한다.
▶️ 만약에 리스트가 비어있지 않다면
=> 마지막 node의 ‘next’를 생성된 node를 가리키도록 하고 tail이 새롭게 생성된 node를 가리키도록
▶️ 새로운 node가 추가되면 길이 +1

pop (끝 제거)

▶️ 리스트가 비어있는지 확인 if(!this.head) return undefined
▶️ 비어있지 않은 경우 tail에 도달할 때 까지 리스트를 따라간다. while(current.next)
▶️ 따라가는 동안 이전 노드를 항상 추적
▶️ 마지막 이전 node의 next를 null로 설정하고, tail값을 이전 값으로 설정
▶️ 길이 -1

shift (맨 앞 제거)

▶️ 현재 head가 가리키고 있는 node 제거
▶️ head를 head가 가리키고 있는 list의 두번째 노드를 가리키도록 한다
▶️ node가 없을 경우 undefined 반환
▶️ node가 존재 할 경우 현재의 head를 변수에 저장하고
▶️ 현재의 head의 next 노드를 가리키도록 head 업데이트
▶️ 길이 -1
